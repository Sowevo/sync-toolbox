# personal_volume Directory Structure (Reference)

这是 personal NAS 数据目录的长期规范，用于管理个人最重要的数字资产。  
目标是做到：

- **清晰**：任何文件都有明确归属，不会“到处都是杂项”
- **安全**：人生资产（照片、证件、存档）与可替代资源（影视、软件）分离
- **可维护**：5 年后回头看仍然能理解，不需要重新整理
- **可扩展**：未来新增内容时有地方可放，不破坏整体结构
- **便于备份**：不同类型数据采用不同备份策略（照片全量、视频分级、ISO冷备）

---

## Design Philosophy（设计原则）

### 1. 内容类型优先
顶层目录按“是什么”划分，而不是按“重要不重要”或“文件大小”。

### 2. 人生资产与资源库分离
- **人生资产**：不可再生（照片、证件、合同、存档）→ 重点备份  
- **资源库内容**：可替代（软件安装包、ISO、字体）→ 冷备即可  

### 3. 原始数据与应用数据分离
- photos/ 是原始照片仓库（底片）  
- photo_library/ 是飞牛图库展示层（相册，可重建）

### 4. `_notes/` 是系统说明书
所有规范与策略都放在 `_notes/`，未来不会迷路。

---

## Backup Strategy Overview（备份策略概览）

| 目录 | 数据性质 | 建议策略 |
|------|----------|----------|
| photos/ | 人生底片，不可再生 | 全量备份 + 云同步 + 快照 |
| docs/ | 证件合同，小但关键 | 全量备份 + 加密副本 |
| docs/游戏相关/游戏存档/ | 小文件但价值巨大 | 高频备份（每日/每周） |
| videos/ | 大体积，可分级 | 重要视频备份，其他冷存 |
| software/系统镜像/ | 可替代资源 | 冷备即可 |
| software/资源素材/ | 可替代资源 | 冷备即可 |
| photo_library/ | 应用展示层 | 可重建，不必重点备份 |
